Automated Pentest Framework
Tutorial

1. Main Framework: 

    1.1 Introduce the target network inside the given text edit area, 

    1.2 Select the type of scan (required), 

    1.3 Select the number of threads the scanning is going to run with (more threads than cores can result in slower results, default 5, minimum 1, maximum 100), 

    1.4 Select "Global timeout" which affect every TCP or UDP connection timeout, given timeout does not leave the scanning phase.

    1.5 Port range can be left "empty" and the scanner will use top 1000 most common TCP/UDP ports, otherwise it can use a varying range of ports such as: single ones: "80", multiple ones: "80,443", array like input: "80-500", hybrid input: "80-500,8080,8088-9000".


 1.5 Checkalive output gives the state the given hosts are in (either online or offline), their operating system as well as their hostname if we have a DNS response.

    1.6 Port scanner output gives us the state the open/filtered scanned ports are in as well as their service and version.

    1.7 Fingerprinter output gives us the technology and version that runs at that time on a port which exposes a http or https service.

    1.8 CVE Finder output is the results of the modules that tests for CVE's. It gives swift information about the host, port, technology, version, CVE name, cvss score and other details.


2. Finder module uses OSINT tools in order to find as much information about the target host from the internet.

    2.1 WHOIS (pronounced as the phrase "who is") is a query and response protocol that is widely used for querying databases that store the registered users or assignees of an Internet resource, such as a domain name, an IP address block or an autonomous system, but is also used for a wider range of other information. 

    


3. CVE Classification is used to find more info about a given vulnerability such as the probability for it to be expolited, the EPSS score and the date it was discovered. The input can be given by enumerating the CVE's names like "CVE-2022-27225,CVE-2022-27223" or be left empty for TOP 100.


4. Fuzzing module is used to use in a fuzzing attack different dicitonaries from our default or by uploading a custom dictionary, the output of our modules is a dirbuster set on "silent" to avoid clutering the output and be given only the necessary information about the process.


5.  History is the module that allows you to see the past activty of the framework and select hosts to be retested. Each host is a link that sends you to the main framework module in order to retry a pentest where you can either override or save a new report of the pentest. The table in history shows us the date and time at which the report has been created, the number of CVE's found on the host, the enumerator results from the "dictionary" module and the open ports found by the port scanner in main framework.
